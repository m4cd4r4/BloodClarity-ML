# Contributing to BloodClarity-ML

Thank you for your interest in contributing to BloodClarity-ML!

## How to Contribute

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature`
3. **Make your changes**
4. **Run tests**: Ensure all tests pass
5. **Commit**: Use descriptive commit messages
6. **Push**: `git push origin feature/your-feature`
7. **Pull Request**: Submit a PR with a clear description

## Areas for Contribution

### High Priority

- **Format Expansion**: Add support for new laboratory providers
- **Language Support**: Multilingual biomarker extraction
- **Validation Rules**: Additional biological plausibility checks
- **Test Cases**: Real-world PDF test samples (de-identified)

### Research

- **Optimisation**: Further model compression techniques
- **Architecture**: Alternative model architectures
- **Evaluation**: Comparative studies with other systems

### Documentation

- **Tutorials**: Step-by-step guides for common tasks
- **Examples**: Integration examples in different frameworks
- **API Docs**: Comprehensive API documentation

## Code Standards

- **Python**: Follow PEP 8
- **TypeScript**: Use ESLint + Prettier
- **Tests**: Add tests for new features
- **Docs**: Update documentation for API changes

## Commit Message Format

```
type(scope): brief description

Detailed explanation (optional)

Closes #issue-number (if applicable)
```

Types: `feat`, `fix`, `docs`, `test`, `refactor`, `perf`, `chore`
